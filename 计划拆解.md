以下是为你量身定制的**12周强化落地计划**，采用**渐进式架构**（从单Agent到多Agent协作），每周都有可运行的代码产出：

## 🗓️ 阶段一：基础架构与核心Agent（第1-4周）

### Week 1: 技术储备与最小可行性验证
**目标**：搭建基础框架，验证LangChain/LangGraph在多角色场景的可行性

- **核心任务**：
  - 确定技术栈：**LangGraph**（状态机编排）+ **Pydantic**（结构化输出）+ **SQLite/Redis**（记忆持久化）
  - 实现基础**Dice Roller工具**（COC 7th版骰子系统：1d100、优势/劣势、大失败/大成功判定）
  - 构建**Prompt模板系统**（角色卡格式、COC世界观约束）
  - **产出**：能进行简单技能检定的命令行Demo

- **面试准备**：
  - 整理Multi-Agent vs Single-Agent架构决策文档
  - 理解Agentic Design Patterns（反思、工具使用、规划）

- **代码目标**：`coc_system/core/dice.py` + `coc_system/prompts/base.py`

### Week 2: KP Agent（主持人）核心实现
**目标**：实现游戏主持人Agent，掌握叙事生成与规则判定

- **核心任务**：
  - 设计**KP Agent状态机**：`准备阶段`→`探索阶段`→`战斗/检定阶段`→`结算阶段`
  - 实现**场景管理模块**：地点状态、可交互对象、时间推进
  - 集成**COC规则引擎**（技能列表、属性计算、SAN值检定）
  - **RAG增强**：加载COC规则书PDF作为知识库（使用Vector Store）
  - **产出**：KP能主持一个简单的密室调查场景

- **面试重点**：
  - ReAct模式实现（推理→行动→观察）
  - 长文本叙事生成的一致性控制技巧

- **代码目标**：`agents/kp_agent.py` + `knowledge_base/rulebook_rag.py`

### Week 3: NPC Agent（非玩家角色）与角色扮演
**目标**：实现有独立人格的NPC Agents

- **核心任务**：
  - 设计**NPC Agent架构**：
    - 独立记忆（短期：本轮对话，长期：关系网、秘密知识）
    - 人格特质（Big Five/OCEAN模型+COC职业特性）
    - 情感状态机（怀疑、恐惧、合作、敌对）
  - 实现**NPC生成器**：根据场景自动生成带秘密、动机、线索的NPC
  - **多模态准备**：NPC头像生成接口（可选，用于GitHub展示）
  - **产出**：3个可互动的NPC，能进行对话、隐藏信息、被说服/威胁

- **技术亮点**：
  - 使用**Reflection Pattern**让NPC回顾并总结互动
  - 实现**情感向量存储**（NPC对玩家的态度变化）

### Week 4: World Agent（世界模拟器）
**目标**：实现环境与物理规则的一致性

- **核心任务**：
  - 构建**World State Graph**：地点拓扑、物品状态、时间线、天气/氛围
  - 实现**环境反馈机制**（玩家行动→世界状态变化→触发新事件）
  - **事件系统**：神话生物出现逻辑、线索刷新机制
  - **物理一致性检查**（防止"穿墙"、物品凭空出现等LLM幻觉）
  - **产出**：一个可探索的1920年代小镇场景，物品可交互

- **架构思考**：
  - World Agent作为**单一事实来源（Single Source of Truth）**
  - 设计Agent间通信协议（Pub/Sub vs 直接调用）

## 🗓️ 阶段二：多Agent协作与Player Agents（第5-8周）

### Week 5: Player Agent（AI玩家）与多Agent通信
**目标**：实现AI玩家，构建Agent间交互协议

- **核心任务**：
  - 设计**Player Agent决策逻辑**：
    - 目标驱动（探索、生存、解谜）
    - 风险厌恶（SAN值管理、战斗vs逃跑决策）
    - 社交策略（与NPC互动的话术选择）
  - 实现**Agent间消息总线**：使用LangGraph的`StateGraph`管理多Agent流转
  - **对话仲裁机制**：当多个Agent同时发言时的轮次管理
  - **产出**：1个AI玩家参与KP主持的模组，能自主探索

- **技术深度**：
  - **Graph-based Multi-Agent**：理解节点（Agent）与边（条件流转）
  - **共享记忆 vs 私有记忆**的设计

### Week 6: 记忆系统与长程叙事
**目标**：解决长团（长期跑团）的一致性问题

- **核心任务**：
  - 实现**分层记忆系统**：
    - 工作记忆（当前场景）
    - 情节记忆（本模组关键事件）
    - 语义记忆（世界观常识、NPC关系网）
  - **记忆压缩**：使用LLM定期总结冗长对话
  - **时间线追踪**：确保事件因果关系正确（防止NPC死而复生）
  - **产出**：能运行超过20轮对话而不丢失关键线索

- **面试价值**：
  - 掌握RAG优化技巧（分块策略、重排序）
  - 理解LLM的上下文限制与缓解策略

### Week 7: 观察者Agent（Observer）与数据收集
**目标**：构建系统的"黑匣子"，用于评估与回放

- **核心任务**：
  - 设计**Observer Agent架构**：
    - 事件日志（结构化JSON：谁在何时做了什么）
    - 叙事文本（自然语言总结）
    - 状态快照（关键决策点的完整World State）
  - 实现**评估指标**：
    - 叙事一致性（检测吃书/设定冲突）

    - 游戏性指标（线索发现率、SAN值消耗曲线）
  - **产出**：可视化时间线（Streamlit/Gradio界面展示剧情发展）

## 🎯 关键技术决策建议

### 1. **架构模式选择**
建议使用 **"Hub-and-Spoke" + "Agent Supervisor"** 混合模式：
- **KP Agent作为Supervisor**：决定当前哪个Agent行动（轮次管理）
- **World Agent作为Source of Truth**：所有Agent通过工具查询/修改世界状态，避免直接修改共享变量

### 2. **记忆实现方案**
```python
# 推荐：分层记忆架构
class AgentMemory:
    short_term: List[Message]      # 当前对话窗口
    working_memory: str            # LLM生成的当前目标/计划
    long_term_store: VectorStore   # 持久化记忆（Chroma/Pinecone）
    episodic_memory: List[Episode] # 关键事件（带时间戳和情感标签）
```

### 3. **COC规则集成**
不要硬编码所有规则，使用**Function Calling**让KP Agent按需查询：
- `query_rule(skill_name)` - 查询技能效果
- `calculate_damage(weapon, target)` - 计算战斗伤害
- `check_sanity_loss(scene_description)` - 判定SAN值损失

### 4. **Observer Agent的数据结构**
设计**结构化日志**便于后续分析：
```json
{
  "timestamp": "1924-03-15T14:30:00",
  "round": 12,
  "agent_type": "KP",
  "action": "scene_description",
  "content": "图书馆里弥漫着霉味...",
  "world_state_delta": {"location.lighting": "dim"},
  "npc_emotion_changes": {"librarian.suspicion": +0.2}
}
```

## ⚠️ 风险控制与建议

1. **复杂度管理**：前4周先不要搞太多NPC，先跑通"1KP+1Player+1World"的最小闭环
2. **成本控制**：使用OpenAI API时设置Layered Model策略（复杂推理用GPT-4，简单响应用GPT-3.5）
