> **掉 SAN 才是终极损失函数。**  
> 我们正在构建一个开源框架，让多个 AI 智能体（Agents）模拟完整的克苏鲁呼唤跑团体验——包括一名守秘人（KP）、一队调查员（4-6人）、各色 NPC、一个监督规则执行的**检察官（Overseer）**，以及一名记录整个疯狂历程的**书记员（Scribe）**。  
> 目标：打造一个沙盒，用于观察 emergent（涌现式）的、既恐怖又搞笑的叙事，同时探索多智能体协作的边界。

---

## 项目概述

**SAN-Loss-Function** 是一个实验性开源框架，旨在通过一组分工明确且互相协作（偶尔也会互坑）的 AI 智能体，完整自动化一场《克苏鲁的呼唤》（第七版）跑团游戏。不同于简单的对话机器人，我们的系统将职责分配给以下专业角色：

- 🧙 **守秘人智能体** – 推动剧情、描述场景、对玩家行动做出反应（但全程被监督）。
- 👥 **调查员智能体**（1-6 名）– 模拟玩家角色，拥有各自 distinct 的性格、技能，以及不断下降的理智值。
- 🎭 **NPC 智能体** – 填充世界：友善的盟友、可疑的镇民、不可名状的恐怖。
- 👁️ **检察官智能体** – 一个元智能体，监控守秘人是否遵循规则，及时指出矛盾或违规（就是那个“规则律师”）。
- 📜 **书记员智能体** – 全程记录会话，生成结构化的日志，既产出可读的故事，也跟踪世界状态。

框架设计为**开源**且**可扩展**，方便研究者、爱好者和游戏设计师实验多智能体叙事。通过大量重复会话，我们希望研究智能体如何协调、在何处崩溃，以及——最重要的——会涌现出怎样意想不到的搞笑或混乱剧情。

**为什么叫“SAN‑Loss‑Function”？**  
在机器学习中，损失函数衡量模型输出与理想目标的差距。而在克苏呼唤里，唯一真正重要的指标就是调查员的理智值掉得有多快。我们将整个叙事过程看作一个优化问题：最小化叙事连贯性？不，是最大化娱乐性和 SAN 值的损失！

## 功能特性（已规划 & 开发中）

- ✅ **多智能体架构** – 模块化的智能体类，每个角色通过共享黑板（blackboard）通信。
- ✅ **COC 第七版规则集成** – 基础技能检定、理智检定、推运、战斗（更多规则持续添加）。
- ✅ **共享世界状态** – 智能体间保持一致的记忆：线索、物品、角色状态、人际关系。
- ✅ **模型无关** – 支持 OpenAI API 和本地模型（Ollama/Llama.cpp）——你可以自由选择 LLM。
- 🚧 **检察官智能体** – 将监控守秘人是否违规，并敢于指出 KP 暗改骰点的行为。
- 🚧 **书记员智能体** – 将生成漂亮的会话日志（Markdown/JSON 格式），附带剧情高光时刻。
- 🚧 **多调查员模拟** – 动态注入个性，让每个调查员智能体都独一无二。
- 🚧 **自动生成模组** – 框架最终能根据克苏鲁神话元素自行创作“一日游”短剧本。
- 🧪 **实验工具包** – 提供并行运行多场会话、收集指标、分析涌现行为的脚本。

## 快速开始（当前状态）

项目正在积极开发中。现在你可以运行一个极简原型（守秘人 + 单个调查员 + 骰子），体验基本交互循环。

### 环境要求
- Python 3.9+
- 安装依赖：
  ```bash
  pip install -r requirements.txt
  ```

### 配置
1. 克隆仓库：
   ```bash
   git clone https://github.com/yourname/SAN-Loss-Function.git
   cd SAN-Loss-Function
   ```
2. 设置 API 密钥（如果使用 OpenAI）：
   ```bash
   export OPENAI_API_KEY="你的密钥"
   ```
   若使用本地模型，请编辑 `config/settings.py` 指向你的 Ollama 端点。

### 运行示例
```bash
python examples/basic_demo.py
```

示例输出：
```
[守秘人] 废弃的精神病院矗立在你面前，铁门在你身后吱呀一声关上了。
[调查员] 我想进行一次“侦察”检定，看看入口附近有没有异常。
[骰子] 掷出 34（目标 60）→ 成功！你在落叶下发现了一本血迹斑斑的日记。
...
```

## 项目结构（持续演化中）

```
.
├── agents/               # 智能体定义：守秘人、调查员、NPC、检察官、书记员
├── core/                 # 核心引擎：黑板、调度器、规则解析器
├── prompts/              # YAML 格式的提示词模板（支持中文）
├── config/               # 配置（API 密钥、模型选择、智能体数量）
├── examples/             # 示例脚本和生成的跑团日志
├── tests/                # 单元测试
├── experiments/          # 批量运行会话、收集指标的脚本
└── requirements.txt
```

## 我们的愿景

我们相信多智能体系统能开启一种全新的互动叙事形式——不可预测、可重玩、创意无限。通过开源这个框架，我们邀请社区：

- **添加新的智能体行为**（比如一个“混沌智能体”，随机引入剧情转折）。
- **集成不同的 TRPG 规则**（D&D、赛博朋克等）。
- **研究涌现叙事**，将本系统用作 AI 对齐或协作研究的试验场。
- **纯粹找乐子**，看 AI 调查员们如何以越来越离谱的方式发疯。
